# Business Document: Order_Items Table

## 1. Business Purpose
The `Order_Items` table represents the details of individual items within an order placed by a customer at a department store. It captures the specifics of each product that is included in an order, such as the quantity ordered and the price at which it was purchased.

## 2. Column Descriptions
- **order_item_id (INTEGER)**: A unique identifier for each order item. It represents the specific instance of a product within an order.
- **order_id (INTEGER)**: A foreign key that references the primary key of the `Customer_Orders` table. It identifies the order to which the item belongs.
- **product_id (INTEGER)**: A foreign key that references the primary key of the `Products` table. It identifies the product that is being ordered.

## 3. Aggregation Methods
- **order_item_id**: Not typically aggregated, but can be used to count the number of items in each order.
- **order_id**: Can be used to aggregate the number of items per order.
- **product_id**: Can be used to aggregate the total quantity of a product sold across all orders.
- **quantity**: (not listed, but would be useful) Can be used to calculate the total quantity sold of a product, average quantity per order, etc.
- **price**: (not listed, but would be useful) Can be used to calculate the total revenue generated from each product or each order.

## 4. Calculable Metrics
- **Average Product Price**: The average selling price of a product, calculated by dividing the total revenue from the product by the total quantity sold.
- **Total Orders**: The total number of orders placed.
- **Total Revenue**: The total revenue generated from all orders.
- **Average Order Value**: The average revenue per order, calculated by dividing the total revenue by the total number of orders.

## 5. Common Filters
- **By Order ID**: To retrieve all items for a specific order.
- **By Product ID**: To retrieve all orders that include a specific product.
- **By Date Range**: To filter items placed within a certain date range.
- **By Customer ID**: To retrieve all items for a specific customer, assuming there is a foreign key in the `Customer_Orders` table linking to the `Customers` table.

## 6. Join Guidance
- **Customer_Orders**: To get additional details about the order such as the customer who placed it, the order date, etc.
- **Products**: To get details about the product, such as the product name, category, and price.
- **Customers**: If needed, to link the order to the customer who placed it, assuming there is a foreign key in the `Customer_Orders` table.

## 7. Query Patterns
### Example 1: Total Revenue by Product
```sql
SELECT Products.product_name, SUM(Order_Items.quantity * Order_Items.price) AS total_revenue
FROM Order_Items
JOIN Products ON Order_Items.product_id = Products.product_id
GROUP BY Products.product_name;
```
**Business Explanation**: This query calculates the total revenue generated by each product, which can help identify the best-selling items.

### Example 2: Average Quantity per Order
```sql
SELECT AVG(Order_Items.quantity) AS average_quantity_per_order
FROM Order_Items;
```
**Business Explanation**: This query provides insight into the average number of items per order, which can be useful for inventory management and understanding customer purchasing patterns.

### Example 3: Total Orders Placed in a Specific Month
```sql
SELECT COUNT(*) AS total_orders
FROM Order_Items
JOIN Customer_Orders ON Order_Items.order_id = Customer_Orders.order_id
WHERE MONTH(Customer_Orders.order_date) = MONTH(CURRENT_DATE) AND YEAR(Customer_Orders.order_date) = YEAR(CURRENT_DATE);
```
**Business Explanation**: This query helps determine the total number of orders placed in the current month, which can be used for performance tracking and forecasting.
